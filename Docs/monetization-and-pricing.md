# Math Dash – Monetization & Pricing

## 1. Freemium Model

- **Free Tier**
  - Access to a small starter set of topics in each category:
    - Multiplication: 2×, 3×, 4× tables.
    - At least one starter division topic.
    - Number bonds: bonds to 10.
    - Doubles/halves: simple doubles/halves up to at least 10.
  - Access to core modes (timed, practice) for those topics.
  - Offline play for all free content.

- **Core Upgrade (One‑Time Purchase)**
  - Unlocks:
    - All remaining standard multiplication tables (5×–12×) and their division topics.
    - Extended number bonds (Make 20, Make 50, Make 100) and more advanced doubles/halves.
    - Adaptive practice mode.
    - Local multiplayer "Dash Duel".
    - Teacher tools for a limited number of classes.
  - Positioned as low‑cost, high‑value, one‑time purchase.

## 2. AI Add‑On – Credit Packs (Post‑MVP)

- Optional, not required to use core app.
- Users buy packs of credits (e.g., 5, 10, 25).
- Each AI analysis (e.g., personalised practice plan, class analysis) consumes credits.
- When credits run out:
  - All non‑AI features continue to work.
  - Users can choose to purchase more credits.

## 3. Pricing Guidelines (Draft)

- **Home License – Core App**
  - Target price (Tier 1 markets: US, UK, EU, Australia, New Zealand):
    - USD $6.99 (United States)
    - GBP £5.99 (United Kingdom)
    - EUR €6.99 (Eurozone)
    - AUD A$10.99 (Australia)
    - NZD NZ$11.99 (New Zealand)
  - Regional pricing later at ~50–70% of Tier 1.

- **Teacher / Small‑School License – Core App**
  - Simple per‑teacher license (Tier 1 markets):
    - USD $19.99 (United States)
    - GBP £17.99 (United Kingdom)
    - EUR €19.99 (Eurozone)
    - AUD A$29.99 (Australia)
    - NZD NZ$32.99 (New Zealand)
  - Future: volume discounts / site licenses once demand validated.

- **AI Credit Packs**
  - Priced according to:
    - Marginal AI compute cost.
    - Perceived value of analyses.
  - Bundles should feel optional but attractive for power users.

## 4. Upgrade UX Requirements

- All locked content clearly marked.
- Tapping locked content:
  - Shows upgrade modal with:
    - Clear explanation of benefits.
    - Pricing outline (or “Learn more”).
    - “Not now” option that returns to previous context.
- All upgrade flows must be adult‑gated and use platform‑native payments.

## 5. Business Model Alignment

- Core app: high‑volume, low‑price, “no‑brainer” purchase for parents/teachers.
- AI: discrete, optional spend that scales with usage.
